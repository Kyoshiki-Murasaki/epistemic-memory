# AI Memory System

Tldr; existing memory tools help the AI answer

> “what do i remember”

this system would also force the AI to answer

> “why should i trust this”
>
> “am i allowed to use this here”
>
> “is this strong enough to act on”
>
> “what else will need to change if this turns out to be wrong”

that is the difference

## 1. What current AI memory tools do

tools like Mem0, Zep and other AI memory systems help an AI remember information across conversations and over time

they can usually do things like

- read conversations and find details that may be useful later
- save things such as names, preferences, goals, decisions and past events
- search through those memories when the user returns
- bring useful information back before the AI answers
- organise information into profiles, timelines or connected facts
- update some memories when new information is added
- keep information separate between different users, sessions or projects
- sometimes keep the original source or show where a memory came from
- sometimes understand that a fact was true before but is no longer true
- sometimes allow developers or users to edit or delete memories

these things are useful and i am not saying existing memory tools do nothing more than save random notes, many of them already try to handle time, updates, sources and different kinds of memory

but even when the memory itself is stored correctly there is still another problem

the AI still has to decide whether that information should actually be trusted and used for the thing it is doing right now

## 2. The problem is not only what the AI remembers

people do not just want an AI that remembers more things

they want an AI that uses the right information, for the right reason, in the right situation

for example a customer may say

> “i already paid for this order”

that statement should be remembered because it may be important, but it should not immediately become

> “the customer has paid”

the first thing is something the customer said, the second thing sounds like a confirmed fact

the billing system may still show that the payment failed, or the payment may still be processing, or the customer may have paid through another method that is not showing yet

so the system should keep both pieces of information clearly

- the customer says they paid
- the billing system does not currently confirm the payment

then it should also know what the AI is allowed to do with that information

the AI may be allowed to say

> “i can see that you said you paid, but the payment is not confirmed in the billing system yet”

but it should not be allowed to mark the order as paid, ship the product or issue another refund only because the customer said it

this is the bigger problem i want to solve

not just remembering information, but controlling what the AI can believe, say and do based on that information

## 3. Different information should not be treated in the same way

right now an AI may receive information from many places

- something the user said
- something another employee said
- something the AI guessed
- something found in an old conversation
- something written in a company document
- something stored in the billing system
- something stored in the CRM
- something confirmed by a manager
- something generated by another AI

all of these sources are not equally reliable

for one task the billing system may be the most trusted source, for another task the user may be the best source

for example the billing system should probably be trusted more when checking whether a payment was completed

but the user should probably be trusted more when saying what name they prefer to be called

the system should allow a company to decide which sources should be trusted for which kinds of decisions

this does not mean the system magically knows what is true

it means the system knows what was said, who or what said it, what other information supports it, what information disagrees with it and which source is allowed to control a certain decision

## 4. Information should have a clear status

the system should not save everything as a simple fact

it should keep clear differences between things such as

- something that was only mentioned
- something the user believes
- something the AI guessed
- something the user is considering
- something the user plans to do
- something someone promised to do
- something confirmed by another source
- something verified by a trusted system
- something that is being disputed
- something that used to be true
- something that was corrected
- something that should no longer be used

for example

> “i may move to London”

should be saved as a possible future plan, not as the users current location

if the person later says

> “i decided to stay in Delhi”

the system should not delete the London statement as if it never happened, because it may still be useful as part of the history

instead it should show that moving to London was considered before, but it is no longer the current plan

the original words should stay saved exactly as they were said, while the systems understanding of those words can change when new information appears

this means the AI can suggest what a statement probably means, but it cannot quietly rewrite the original statement or turn a guess into a confirmed fact

## 5. The information should only be used when it fits the current task

information can be correct and still be wrong to use

for example

> “the user likes pixel art”

may be useful when designing the users personal game or profile page

but it should not automatically affect every website the AI builds for them

the website may be for a bank, a hospital or another customer where the users personal taste does not matter

the system should check

- who the AI is helping
- what task the AI is doing
- what decision is being made
- whether the information is relevant to that task
- whether the AI is allowed to access it
- whether the information is private
- whether the information is current enough
- whether the source is trusted enough for that decision

so memory would not just be searched by asking

> “is this information related to the conversation”

it would also ask

> “should this particular AI be allowed to use this information for this particular task”

## 6. The system should track promises and unfinished work

AI memory systems usually focus on facts about a person or things that happened in the past

but AI agents will also need to remember promises, responsibilities and unfinished actions

for example

- the company promised to send a refund within five days
- the customer needs to upload one more document
- the account manager said they would call on Friday
- the support agent must wait for approval before closing the case
- engineering is still investigating the problem
- the customer asked not to be contacted by phone
- a manager needs to review an exception

these should not be stored as normal facts and then forgotten inside a long list of memories

the system should separately track

- who is responsible
- what they promised to do
- who they promised it to
- when it needs to be done
- what must happen before it can be completed
- whether it is still waiting, completed, cancelled or overdue
- what proof shows that it was completed

this would help stop AI systems from making promises and then forgetting them

## 7. When something changes the correction should spread everywhere

changing one memory is not enough if the old information has already been used somewhere else

for example imagine the system wrongly decided that a customer was allowed to receive a refund

that information may already have been used in

- a summary of the customer
- a support reply
- a plan created by the AI
- a refund waiting to be processed
- a message sent to another employee
- a report
- another saved memory

when the information is corrected the system should find the other things that were based on it

it should be able to

- mark the old information as no longer usable
- update summaries that included it
- stop actions that have not happened yet
- warn people about actions that may have been affected
- show which decisions were based on the wrong information
- allow the correction to be checked and reversed if needed
- keep the old version as history instead of pretending it never existed

so the system would not only update what the AI remembers, it would also deal with the effects of the wrong memory

## 8. Before an important action the system should check the information again

finding a memory should not automatically give the AI permission to act on it

before the AI does something important the system should check

- where the information came from
- whether it is still current
- whether another source disagrees with it
- whether it has been confirmed
- whether the source is trusted for this kind of decision
- whether the AI is allowed to use it
- whether a human needs to approve the action
- whether the information is strong enough to support the action

for example the AI may be allowed to use an unconfirmed statement to ask the user a follow up question

but it may not be allowed to use the same statement to send money, close an account, change an address or approve a refund

the more serious the action is, the stronger the information should need to be

## 9. People should be able to understand why the AI did something

when the AI gives a wrong answer it should not be enough to say

> “these memories were retrieved”

people should be able to see the full reason

for example

- the AI believed the customer could receive a refund
- it believed this because of a certain support message
- the support message was treated as confirmed because of a certain rule
- that information was used instead of older information
- the refund rule allowed the AI to continue
- if that memory had not been used the refund would not have been approved

the system should show

- which information affected the answer or action
- where that information came from
- how the system understood it
- what other information disagreed with it
- why one source was trusted over another
- what rule allowed the information to be used
- what would have happened if that information was removed or corrected

this would help developers understand whether the problem came from the original information, the AIs interpretation, an incorrect rule, an outdated source or the final AI answer

## 10. People should be able to inspect and correct what the system believes

users and developers should be able to see what the system currently believes about a person, company, project or situation

they should be able to see

- the original statement
- the systems current understanding
- whether it is confirmed or uncertain
- where it came from
- when it was added
- when it was last checked
- where it has been used
- whether another memory disagrees with it
- what would change if it was corrected or deleted

when the system is not sure what something means it should be able to ask for help instead of making the decision silently

for example

old information

> the customer lives in Delhi

new statement

> “i will probably move to London next month”

suggested result

> keep Delhi as the current location, save London as a possible future plan, do not change the current address yet

a user or employee could then approve, change or reject that suggestion

## 11. The structural difference

most memory systems are mainly built around this question

> “what information should we save and bring back later”

this system would be built around a different question

> “what information is this AI allowed to trust and use for this exact task or decision”

the system would keep two separate things

### what actually happened

the original messages, documents, system records and events would stay saved without being silently rewritten

### what is currently supported

the system would maintain its current understanding of the situation, including what is confirmed, uncertain, disputed, outdated or replaced

when the current understanding changes the original history would still remain

this means the system can change its mind without changing the past

## 12. What would make this different from existing memory tools

the main difference is not that this system remembers more information, and it is not just that it remembers information for longer

the difference is that it controls whether remembered information should actually be used

every important piece of information would keep its original source, its current status, when it was true, what disagrees with it and what tasks it is allowed to affect

different sources would have different levels of trust depending on the decision being made

the system would separately track facts, guesses, plans, promises, responsibilities and unfinished actions

when something changes it would find the summaries, decisions and pending actions that depended on the old information instead of only editing one saved memory

before the AI takes an important action the system would check whether the information is current, supported, allowed and strong enough to act on

after the AI takes an action people would be able to see exactly which information and rule caused it

so the goal is not only to help AI remember

the goal is to stop AI from making decisions using information that is old, unsupported, conflicting, private, taken from the wrong source or not relevant to what the AI is doing
